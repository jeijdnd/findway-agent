## 一、产品概述

### 1.1 产品定位

**标识Agent** 是一个面向建筑导视标识设计全流程的智能助手，定位为**设计师的工作台 + 第二大脑**。

核心设计哲学：**你不是从零生成，你是在已有资产上编排**。标识设计师的真实工作流是以"套用旧项目"为核心的——Agent的价值在于帮助你更快地找到该套用哪个旧项目、更快地完成套用后的修改、更准确地对比两版清单的差异。

### 1.2 真实工作流（已对齐）

```
┌──────────────────────────────────────────────────┐
│  项目进来（时间极紧，内部原因导致压缩）              │
├──────────────────────────────────────────────────┤
│                                                    │
│  第①步：出第一版清单                                │
│    → 根据平面图确定需要哪些标识                      │
│    → 款式全部从旧项目套用（按建筑类型匹配）           │
│    → 产出：预算/概算用的标识清单                     │
│                                                    │
│  第②步：设计部门出部分方案                           │
│    → 只出"重要牌子"的设计方案                        │
│    → 普通牌子可能没有方案就直接跳过                   │
│                                                    │
│  第③步：出施工图 + 布点图                           │
│    → 有方案的牌子 → 自己画                          │
│    → 没方案的牌子 → 套用其他项目的旧施工图            │
│    → 图框需要替换项目信息                            │
│    → 布点图在兔钉网(twoding.net)中制作               │
│                                                    │
│  第④步：出第二版清单                                │
│    → 基于施工图重新统计                              │
│    → 与第一版会有款式/数量的差异                     │
│                                                    │
│  第⑤步：打包交付                                     │
│    → 布点图 + 施工大样图 + 清单                      │
│    → 给施工方（深化用）或概算部门（算钱用）           │
│                                                    │
│  第⑥步：配合调整                                     │
│    → 施工方/业主提变更 → 改图 → 改清单               │
│    → 多项目并行，反复跳跃切换                         │
│                                                    │
└──────────────────────────────────────────────────┘
```

### 1.3 核心痛点 vs Agent介入点

| 痛点 | 现状 | Agent怎么帮 |
|------|------|------------|
| 旧项目选型靠记忆 | "XX学校跟这次差不多"然后去翻文件夹 | 根据项目特征（类型/规模/楼栋）智能推荐最匹配的旧项目 |
| 套用后修改量大 | 图框逐张改、款式逐行调 | 批量替换图框信息；差异对比只改变化的 |
| 两版清单重复做 | 施工图画完重新统计，不知道哪些变了 | 自动对比前一版vs施工图版，只让你确认变化项 |
| 设计方案只覆盖部分 | 不确定哪些牌子需要自己画 | 方案覆盖率分析：哪些有方案、哪些要套用 |
| 多项目并行混乱 | 七八个项目同时跑，忘记哪个做到哪步了 | 项目仪表盘，快速切换到不同项目的当前状态 |
| 规范懒得查 | 凭经验，偶尔翻一下但很慢 | 自然语言秒查 GB/JGJ/CJJ/CECS/DB |

### 1.4 用户画像

- **主要用户**：标识设计师（建筑导视设计方向）
- **核心工具链**：Excel（清单）、AutoCAD 2020 + 格原协同平台插件（施工图/DWG）、兔钉网 twoding.net（布点图）
- **工作特征**：项目时间极紧、高度依赖旧项目套用、多项目并行反复跳跃
- **技术背景**：非程序员，熟练CAD/Excel操作，可接受桌面应用式交互
- **特殊约束**：公司微信存在监控风险，数据应保留在本地

## 二、核心功能模块

### 模块A：旧项目智能匹配与套用（优先级：P0 ★★★★★）

> 这是Agent最基础、最刚需的能力。比母版库更底层。

#### A-1 旧项目索引与检索

**功能描述**：扫描本地旧项目目录，建立可检索的项目指纹索引。

**索引内容**：
- 项目名称、类型（学校/办公/住宅/实验室/...）
- 楼栋数量与类型（教学楼×3、宿舍×2...）
- 标识款式清单指纹（A类X项、B类Y项...）
- 文件路径（清单Excel路径、施工图CAD目录路径）
- 完成时间

**检索方式**：
- 输入新项目特征 → 返回匹配度排序的旧项目列表
- 支持模糊匹配："带地库的学校项目""有实验楼的学校"
- 匹配维度：项目类型 > 楼栋结构相似度 > 规模 > 时效性

#### A-2 款式套用助手

**功能描述**：选中旧项目后，快速将其清单中的款式复制到新项目清单。

**操作流程**：
1. 选择匹配的旧项目（或Agent推荐TOP 3）
2. 预览旧项目的款式清单（分类浏览）
3. 勾选需要套用的款式（支持全选/按类选/逐个选）
4. 基于新项目的平面图信息调整数量
5. 生成新项目第一版清单

**与母版库的关系**：
> 母版库不是一次建成的，而是从旧项目套用过程中**逐步沉淀**出来的。当你多次从不同旧项目套用了相似的款式组合后，Agent会自动将其提炼为"推荐款式集合"，这就是母版库的雏形。

### 模块B：清单差异对比（优先级：P0 ★★★★★）

#### B-1 两版清单对比

**功能描述**：比较前后两版标识清单，精确识别变化项。

**典型场景**：
- 第一版（概念清单） vs 第二版（施工图清单）
- 设计变更后清单 vs 变更前清单
- 施工方深化后清单 vs 原清单

**对比维度**：

| 变化类型 | 判定条件 | 标注 |
|----------|----------|------|
| 新增 | 第二版有、第一版无的款式 | 🟢 绿色 |
| 删除 | 第一版有、第二版无的款式 | 🔴 红色 |
| 修改 | 同一款式下数量/尺寸/材质任一变化 | 🟡 黄色（标注具体变化字段） |
| 不变 | 完全一致 | ⚪ 无标注 |

**输出**：
- 差异报告（Excel格式，颜色标注）
- 变化汇总卡片（"新增12项，删除3项，修改8项，不变47项"）
- 一键应用到目标清单

### 模块C：CAD施工图辅助（优先级：P1）

#### C-1 DWG信息提取

**功能描述**：读取DWG图纸，提取标识相关结构化信息。

**提取内容**：
- 图框中的项目信息文字
- 标识图例图块列表（名称、图层、引用数）
- 图纸目录结构（哪些图纸含标识）
- 兼容格原协同平台插件（需处理文字样式导致中文问号的问题）

#### C-2 图框批量替换（Phase 2+）

**功能描述**：给定新项目信息，批量替换DWG中所有图框的文字。

**安全机制**：
- 替换前自动备份到 `.bak` 目录
- 预览变更内容（替换前文字 → 替换后文字），用户逐项确认
- 支持撤销回滚

#### C-3 图例批量替换（Phase 2+）

**功能描述**：设计方案更新后，批量替换DWG中的标识图例图块。

### 模块D：项目仪表盘与多项目并行管理（优先级：P1）

#### D-1 项目总览面板

**功能描述**：桌面应用主界面，展示所有进行中项目的当前状态。

**每个项目卡片显示**：
- 项目名称 + 类型标签
- 当前阶段（清单阶段 / 施工图阶段 / 交付阶段 / 配合阶段）
- 最近操作时间
- 关键文件快捷入口（清单Excel / CAD目录 / 布点图链接）
- 待办提醒（如"第二版清单还未做""图框信息未替换"）

#### D-2 项目快速切换

**功能描述**：一键切换到不同项目的工作上下文。

**切换内容**：
- 加载该项目的文件路径配置
- 显示该项目的操作历史
- 恢复上次未完成的操作状态

### 模块E：国家标准规范查询（优先级：P1）

#### E-1 自然语言规范检索

**覆盖标准**：GB / JGJ / CJJ / CECS / DB 系列

**查询方式**：自然语言输入，如"学校卫生间标识的安装高度要求"

**输出格式**：
- 标准编号 + 条文编号
- 全文引用
- 关键参数提取
- 与当前项目的适用性判断

### 模块F：母版库渐进式构建（优先级：P2）

#### F-1 自动沉淀

**功能描述**：Agent观察你的套用行为，自动将高频套用的款式组合沉淀为母版模板。

**触发条件**：同一款式组合在≥3个不同项目中被套用时，Agent提示"这个组合似乎经常用到，要加入母版库吗？"

#### F-2 母版库管理

**功能描述**：统一管理沉淀下来的款式模板，支持增删改查和分类。

### 模块G：AI对话助手（优先级：P0 ★★★★★）

> 这是标识Agent的**交互核心**。所有功能模块都可以通过对话触发，也可以在右侧面板手动操作。

#### G-1 对话界面

**功能描述**：软件左侧为AI对话区，参考主流AI软件（ChatGPT、微信等）的聊天形态。

**界面特征**：
- 消息气泡形式，用户消息靠右（蓝色），AI回复靠左（灰色）
- 支持多轮对话，上下文连续
- 打字动画效果（逐字显示AI回复）
- 对话历史持久化，关闭重开不丢失
- 支持输入框 + 回车发送 / 按钮发送

#### G-2 对话驱动操作

**功能描述**：用户通过自然语言对话触发所有功能，无需记忆菜单和按钮位置。

**典型对话示例**：

```
用户："帮我创建一个新项目，珠海理工，学校类型"
AI：  "好的，已创建"珠海理工"项目。需要我帮你找类似的旧项目来套用吗？"
      [右侧面板自动切换到项目详情]

用户："学校卫生间标识的安装高度有什么规范？"
AI：  "根据GB 50763-2012《无障碍设计规范》第3.9.2条：无障碍卫生间标识牌
      安装高度宜为1.40m-1.60m。详见：...(条文引用)"
      [右侧面板自动切换到规范查询结果]

用户："帮我对比珠海理工的两版清单"
AI：  "请上传两份清单文件，或直接拖入下方区域"
      [右侧面板切换到对比页面，自动弹出文件选择]
```

**对话可驱动的操作清单**：
- 创建/查看/删除项目
- 搜索旧项目
- 对比清单（说"对比xx的清单"）
- 查询规范（说"xx规范要求是什么"）
- 提取CAD信息（说"读取这个DWG的图框信息"）
- 修改配置（说"关掉CAD辅助模块"）
- 批量操作（说"帮我把所有学校的项目列出来"）

#### G-3 对话与面板联动

**功能描述**：对话和右侧面板实时联动，互为辅助。

| 用户操作 | 对话区反应 | 面板反应 |
|----------|-----------|----------|
| 对话中说"创建项目" | AI回复确认 | 自动弹出新建表单 |
| 面板上创建了项目 | AI说"已创建xx项目" | 显示新建的项目卡片 |
| 拖入文件对比 | AI报出对比结果摘要 | 显示差异详情表格 |
| 点击面板的功能按钮 | AI提示"需要我帮什么？" | 切换到对应功能页 |

## 四、开发者控制台（自主更新界面）

### 4.1 设计目标

用户不需要每次调整都依赖开发人员或重新对话。通过控制台可以：

| 操作 | 说明 | 示例 |
|------|------|------|
| **增减功能模块** | 像装插件一样开关Phase 1/2/3的功能 | 暂时不需要规范查询 → 关闭模块C |
| **调整工作流步骤** | 修改默认的项目创建流程顺序 | 有些项目不需要布点图 → 跳过该步骤 |
| **修改匹配规则** | 调整旧项目推荐的权重和规则 | "学校类优先匹配近2年的项目" |
| **自定义模板** | 改清单Excel的列结构、加新字段、改格式 | 新增"单价"列、"安装方式"列 |

### 4.2 控制台形态

- 桌面应用中内嵌的"设置/配置"面板
- 可视化配置（开关、下拉选择、权重滑块）+ JSON配置文件高级模式
- 配置变更即时生效，无需重启

### 4.3 配置文件结构（示例）

```json
{
  "modules": {
    "project_matching": true,
    "list_compare": true,
    "cad_assist": true,
    "spec_query": false,
    "master_library": true
  },
  "workflow": {
    "new_project_steps": [
      "collect_info",
      "match_old_project",
      "generate_list_v1",
      "replace_frame",
      "remind_list_v2"
    ]
  },
  "matching_rules": {
    "type_weight": 0.5,
    "structure_weight": 0.3,
    "time_weight": 0.2,
    "max_results": 5
  },
  "list_template": {
    "columns": ["编号","名称","参考尺寸","参考材质","是否带电","单位","数量","备注"]
  }
}
```

## 六、数据流设计（基于真实工作流）

### 6.1 新项目进来

```
用户输入：项目类型 + 楼栋列表 + 平面图信息
         │
         ▼
Agent扫描旧项目目录 → 建立/更新索引
         │
         ▼
根据项目特征匹配旧项目 → 推荐 TOP 3
```

**左右分区说明**：

| 区域 | 占比 | 内容 | 行为 |
|------|------|------|------|
| 对话区（左） | ~40% | 消息列表 + 输入框 | 滚动查看历史，底部输入新消息 |
| 操作面板（右） | ~60% | 仪表盘/匹配/对比/设置 | 对话触发操作时自动切换对应面板 |
| 可拖拽分界线 | — | — | 用户可以拖动调整左右比例 |

### 7.2 交互原则

- **先搜索再创建**：任何操作前，Agent先检查是否有可套用的旧项目
- **预览再确认**：涉及文件修改时，先预览变化再让用户确认
- **增量优于全量**：清单更新优先做差异合并，不覆盖
- **操作可追溯**：记录每次操作日志，支持回退
- **多项目不混淆**：每个项目的文件路径、操作历史独立管理

## 八、非功能性需求

### 8.1 安全性

- 所有数据存储在本地，不上传云端（AI推理除外）
- 文件修改前自动备份到 `.signage-agent/backups/`
- CAD写入操作必须用户逐项确认
- API仅监听 `127.0.0.1`，不暴露到局域网

### 8.2 可靠性

- 旧项目索引自动更新（检测到新文件时增量刷新）
- 清单对比结果标注匹配置信度
- 操作失败自动回滚到备份

### 8.3 性能

- 旧项目扫描（100+项目目录）：< 30秒首次，增量 < 5秒
- 清单差异对比（千行级）：< 10秒
- DWG单文件读取：< 30秒
- GUI操作响应：< 500ms

### 8.4 可扩展性

- 功能模块化，控制台可开关
- 母版库/规范库/索引文件均采用开放格式（JSON/Markdown/Excel）
- API层预留团队扩展接口

## 九、技术选型建议

| 层级 | 技术选项 | 理由 |
|------|----------|------|
| 桌面GUI | Electron + React / Tauri + Vue / 本地Web | 本地运行、跨平台 |
| 后端Runtime | Python (FastAPI) | 与openpyxl/cad处理工具链天然兼容 |
| AI对话 | LLM API（OpenAI兼容 / WorkBuddy API） | 对话意图解析 + 自然语言回复 |
| Excel处理 | openpyxl | 非破坏性读写，保留格式 |
| DWG读取 | cad-viewer skill 或 ezdxf | 先验证可用性 |
| AI编排 | LLM API | 对话意图识别 + 工具调用调度 |
| 本地数据库 | SQLite（可选）+ JSON文件 | 轻量 |
| 打包分发 | PyInstaller 或 Electron打包 | 一键安装 |
